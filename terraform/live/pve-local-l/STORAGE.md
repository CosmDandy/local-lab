# Хранилище хоста `pve-local-l-01` (10.0.1.101)

Заметка о решении по дискам. Тестовый стенд, сохранность данных не приоритет —
приоритет ёмкость под несколько кластеров и обучение.

## Железо

4 × SSD ~500 ГБ:

| Диск | Модель | Назначение |
|------|--------|------------|
| `sda` | Samsung SSD 870 EVO 500GB | Система Proxmox (LVM `pve`: root, swap, thinpool `local-lvm`) + LXC 200 (реестр-кэш) |
| `sdb` | Transcend TS500GSSD220Q | ZFS pool `tank` |
| `sdc` | Seagate BarraCuda ZA500CM10002 | ZFS pool `tank` |
| `sdd` | Seagate BarraCuda ZA500CM10002 | ZFS pool `tank` |

RAM хоста: 188 ГБ (ZFS ARC не лимитирован — памяти с запасом).

## Текущее решение — Путь 1 (ВРЕМЕННОЕ)

Без переустановки Proxmox. Три свободных диска объединены в **striped** (RAID0) ZFS-пул:

```bash
zpool create tank sdb sdc sdd      # ~1.35 ТБ, без избыточности
```

- Все VM-диски новых кластеров (системные + data-диски Longhorn) провижатся из `tank`.
- Система Proxmox и LXC 200 (pull-through registry cache) остаются на `sda` — не трогаем.
- Longhorn: каждому worker-у отдельный data-диск из `tank` под `/var/lib/longhorn`,
  `replicaCount: 2`.

### Сознательный компромисс
Все worker-диски лежат на одном физическом пуле → Longhorn `replica=2` защищает только
от падения ноды/VM (ребут, краш, drain), **НЕ** от смерти физического диска. Striped-пул:
смерть любого из 3 дисков = пул мёртв = стенд пересобирается. Для тестового стенда — ок.
